🛠️ Нове ТЗ: "Personal Finance Tracker" (Трекер фінансів)
Цього разу ми не просто пишемо код, а проектуємо структуру з 3 файлів.

📂 Структура проєкту:
models.py — тут лежить клас Transaction.

utils.py — тут функції для роботи з файлом (save/load).

app.py — головний файл з меню.

📝 Технічні вимоги:
Клас Transaction (models.py):

Атрибути: amount (сума), category (їжа, розваги, оренда), type (Дохід/Витрата).

Модуль utils.py:

Функція calculate_balance(transactions): приймає список і повертає фінальну суму.

Функція filter_by_category(transactions, category): повертає лише транзакції певної категорії (використай List Comprehension).

Головний файл app.py:

Має імпортувати класи та функції: from models import Transaction та import utils.

Користувач може додавати транзакції, бачити баланс та фільтрувати витрати по категоріям.

💡 Нова фішка: __name__ == "__main__"
В кожному професійному файлі Python ти побачиш в кінці таку конструкцію:

Python
if __name__ == "__main__":
    # Код тут запуститься, ТІЛЬКИ якщо ти запускаєш цей файл напряму
    # А не якщо ти його просто імпортував в інший файл
    main_loop()
